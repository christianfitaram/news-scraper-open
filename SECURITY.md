# Security Policy

## Reporting a Vulnerability

If you discover a security issue, please do not open a public issue with exploit details.

Report it privately to the maintainer with:

- a short description of the issue
- affected files or components
- reproduction steps
- impact assessment if known

Until a dedicated private reporting channel is published, contact the maintainer directly through the repository profile or associated contact information.

## Scope

Please report issues related to:

- secret handling
- unsafe webhook behavior
- dependency vulnerabilities
- insecure defaults
- injection risks in request handling or external-service integrations

## Out of Scope

The following are generally out of scope unless they create a direct security impact:

- scraper breakage due to markup changes
- model quality issues
- ordinary rate limiting or upstream API failures
- environment misconfiguration on a local machine
